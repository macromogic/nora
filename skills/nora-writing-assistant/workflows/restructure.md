# Workflow: Restructure

Use this workflow to reorganize a passage or section — paragraph order, topic sentences, transitions — without changing its technical content.

## Goal

The same claims and evidence, organized so the argument is easier to follow.

## Procedure

### 1. Establish scope

Identify the section in scope (typically a few paragraphs to a subsection — not a whole manuscript in one pass unless the user explicitly asks for that and confirms the scope).

### 2. Diagnose the current structure

Before proposing a new structure, identify concretely what's wrong with the current one: buried lead, redundant paragraphs, claims and their evidence separated, weak/missing transitions, inconsistent ordering (e.g. results before the method that produced them). Say what you found before proposing a fix.

### 3. Propose a restructuring

- Reorder, merge, or split paragraphs as needed; rewrite topic sentences and transitions to fit.
- Do not change what is claimed, drop evidence, or introduce new claims to smooth a transition.
- Keep every citation attached to the same claim it was originally attached to — if a claim moves, its citation moves with it.
- If the reorganization would change how strongly something reads (e.g. moving a caveat further from its claim weakens it in practice even without wording changes), call this out explicitly per Writing rule 6.

### 4. Output

Present the proposed structure (e.g. as an outline or a before/after paragraph order) and the revised text. Do not apply changes to the actual file unless the user confirms.
